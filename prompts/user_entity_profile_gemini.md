# 自检与重写提示词

你刚刚为《剑来》已导入章节范围内的一个实体生成了一份 dossier 初稿。现在请对照以下质量规则进行自检，并输出重写后的最终版本。

## 自检规则

逐条检查，如违反则在重写中修正：

1. **字数**：
   - `identity_summary`：80–160 字？
   - `display_summary`：200–440 字？
   - `long_description`：360–840 字，3–6 段？
   - `story_function`：80–180 字？
   - `phase_arc`：120–260 字？
   - `relationship_clusters`：每项 40–100 字，3–6 项？
2. **禁止复述**：`long_description` 的开头是否直接复述了首条 evidence 原文？如果是，必须改写。
3. **禁止模板骨架**：是否用了"在已导入文本中"、"在现有材料里"、"人物关系上"、"从阶段走势看"这类固定过渡？如果超过两处使用同类骨架，必须打散重组。
4. **禁止事件堆砌**：正文是否主要由 turning point 标题拼接而成？如果是，必须改为观点性评述。
5. **覆盖度**：是否覆盖了身份/处境、主要关系、阶段变化、叙事功能中的至少三类？
6. **语料越界**：是否包含 packet 中未提供的信息？
7. **风格**：读起来像"读完已导入章节后的文学评论"，还是像"自动生成的摘要卡"？

## 输出要求

直接返回修正后的完整 JSON，格式与初稿相同。不要附加 markdown 代码围栏、不要附加解释。如果初稿已全部合格，原样返回即可。
