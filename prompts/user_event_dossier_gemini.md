# 事件 Dossier 自检与重写提示词

你刚刚为《剑来》已导入章节范围内的一个事件生成了一份 dossier 初稿。现在请对照以下质量规则进行自检，并输出重写后的最终版本。

## 自检规则

逐条检查，如违反则在重写中修正：

1. **字数**：
   - `identity_summary`：40–120 字？
   - `display_summary`：120–300 字？
   - `long_description`：250–600 字，3–5 段？
   - `story_function`：50–140 字？
   - `relationship_impact`：60–180 字？
2. **禁止复述**：`long_description` 的开头是否直接复述了 description 原文？如果是，必须改写。
3. **禁止模板骨架**：是否用了"在已导入文本中"、"在现有材料里"、"从叙事角度看"这类固定过渡？如果超过两处使用同类骨架，必须打散重组。
4. **禁止参与者堆砌**：正文是否主要由参与者姓名 + 行为的简单拼接组成？如果是，必须改为观点性评述。
5. **覆盖度**：是否覆盖了事件核心内容、参与者抉择、叙事意义、故事线位置中的至少三类？
6. **语料越界**：是否包含 packet 中未提供的信息？
7. **风格**：读起来像"读完已导入章节后的文学评论"，还是像"自动生成的事件摘要卡"？
8. **relationship_impact**：是否具体涉及了参与者之间的关系变化？是否空泛？

## 输出要求

直接返回修正后的完整 JSON，格式与初稿相同。不要附加 markdown 代码围栏、不要附加解释。如果初稿已全部合格，原样返回即可。
